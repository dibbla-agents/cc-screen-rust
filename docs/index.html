Terminal sessions for your AI coding agents, attached from anywhere.

cc-screen runs claude, codex, gemini and kimi as long-lived PTY sessions on one host, and streams them to a phone PWA or a native terminal client — live, over your tailnet. No tmux. No auth. The network is the boundary.

drives  claude · codex · gemini · kimi  — mac & linux, both arches

One engine. Two clients. Zero tmux.

Two clients, one wire

A React PWA — embedded in the server binary — and ccs, a native multi-pane terminal client, speak the same WebSocket contract.

In-process PTY engine

Each session owns its pseudo-terminal for life. A screen model plus a raw-byte replay ring repaint any client cleanly on reattach.

Tailnet-only by design

Agents run with --dangerously-skip-permissions; there's no auth and the server never binds a public interface. Tailscale is the boundary.

Resume on restart

A redeploy ends the agents; auto-restore relaunches each with its CLI's resume flag, conversation intact.

Getting started

Stand the server up on the host where your agent CLIs live; install the client on every device you drive it from. Both ship as checksum-verified binaries for macOS and Linux.

① On your host — the server

Needs Tailscale running, plus the agent CLIs (claude, codex, …) installed & logged in on the host.

$ curl --proto '=https' --tlsv1.2 -LsSf \
    .../releases/latest/download/cc-screen-rust-installer.sh | sh
$ cc-screen-rust install   # systemd / launchd · binds tailnet IP:8839

Then open http://<tailnet-ip>:8839 and Add to Home Screen.

② On your devices — the client

Runs on any Mac or Linux box on the tailnet. Lands in ~/.local/bin — open a new shell if it isn't on your PATH.

$ curl --proto '=https' --tlsv1.2 -LsSf \
    .../releases/latest/download/cc-screen-tui-installer.sh | sh
$ ccs --server http://<host>:8839

A tmux-style client: session switcher, pane grid, Ctrl-A prefix. Or just open the URL — the PWA is built in.

How the bytes flow

phone PWAccs (TUI)
│  WebSocket · raw PTY bytes · over Tailscale, no auth
cc-screen-rust  — PTY engine · screen model + replay ring · broadcast fan-out
│  spawns as PTY children on the host
claude · codex · gemini · kimi

One authoritative engine on the host; clients are thin terminals fed raw output and sending back keystrokes. The wire contract lives in one shared crate, so the PWA and ccs never drift.
