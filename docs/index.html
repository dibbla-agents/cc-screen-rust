▸ how it works

One engine. Two clients. Zero tmux.

  1. 01

    Written in Rust

    One static binary holds the PTY engine, the wire protocol, and the embedded web UI. No runtime to install, no tmux to wrangle.

  2. 02

    Two clients, one wire

    A React PWA built into the binary, and ccs — a native multi-pane terminal client — speak the same WebSocket contract from one shared crate.

  3. 03

    Resume on restart

    Each session owns its PTY for life; a redeploy ends the agents and auto-restore relaunches each with its CLI's resume flag, conversation intact.

▸ getting started

Install in two steps

① The server

On the host where your agent CLIs (claude, codex, …) are installed.

$ curl --proto '=https' --tlsv1.2 -LsSf \
    .../cc-screen-rust-installer.sh | sh
$ cc-screen-rust install   # systemd / launchd · :8839

Then open http://<host>:8839 in any browser.

② The client

On any Mac or Linux box. Lands in ~/.local/bin.

$ curl --proto '=https' --tlsv1.2 -LsSf \
    .../cc-screen-tui-installer.sh | sh
$ ccs --server http://<host>:8839

A tmux-style session switcher and pane grid — or just use the browser.
